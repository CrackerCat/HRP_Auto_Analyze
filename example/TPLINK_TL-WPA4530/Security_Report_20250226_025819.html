漏洞 1: system

CWE-78 CVSS: 7.2 高危

漏洞描述

通过调用`system`函数执行操作系统命令，存在命令注入漏洞。攻击者可以通过构造特定的输入参数来执行任意命令。

触发流程

通过构造特定的输入参数，例如 `argv[1]` 和 `argv[2]`，攻击者可以在`main`函数中触发`wifi_send_msg`或`wifi_exec_cmd`函数。
在`wifi_send_msg`函数中，如果`a1`参数为4，则会调用`wifi_exec_cmd`函数。
在`wifi_exec_cmd`函数中，`vsprintf`函数用于格式化命令字符串，然后通过`system`函数执行该命令。
攻击者可以通过构造包含恶意命令的字符串来执行任意命令。
在`wifi_get_all_config`函数中，`wifi_get_wifi_config`函数会根据不同的配置类型（如3或1）读取并解析配置文件。
这些配置文件的内容可能会被`system`函数执行。
在`wifi_get_wifi_config`函数中，`cJSON_Parse`函数用于解析配置文件。
如果解析失败，`system`函数会被调用来记录错误日志。
在`wifi_get_acl_config`、`wifi_get_guest_config`、`wifi_get_channel_info`、`wifi_get_status_info`、`wifi_get_parentCtrl_config`等函数中，同样存在通过`system`函数执行命令的风险。
在`loadRulesToKernel`函数中，`sendDeviceMsgToKernel`函数会发送设备消息。
这些消息的内容可能会被`system`函数执行。
在`sendCleanMsgToKernel`函数中，`sendNetlinkMsg`函数会发送清理消息。
在`sendGlobalMsgToKernel`函数中，`sendNetlinkMsg`函数会发送全局消息。
在`wifi_log_format`函数中，`snprintf`函数用于格式化日志字符串，然后通过`system`函数执行该命令。
攻击者可以通过构造包含恶意命令的日志字符串来执行任意命令。

漏洞 2: popen

CWE-78 CVSS: 7.5 高危

漏洞描述

通过popen函数执行外部命令时，如果命令字符串未经充分验证或过滤，可能导致命令注入攻击。

触发流程

攻击者可以通过构造恶意的命令字符串，利用未过滤的输入参数，导致执行任意命令。例如，如果`a1`参数直接来源于用户输入，则攻击者可以构造如`"rm -rf /"`这样的命令，从而删除文件系统中的重要文件。
